Phone Latency Tester

Overview

Hardware/software system for measuring end-to-end touch-to-audio latency in mobile devices. Built on the Bela embedded audio platform using C++, with an analog accelerometer detecting the physical tap event and audio analysis capturing the device's speaker response; achieving approximately 226 µs measurement resolution.

Key Specs

  • Platform: Bela (BeagleBone Black-based embedded audio)
  • Language: C++ (real-time audio/sensor acquisition)
  • Sensor: Analog accelerometer for tap detection
  • Measurement resolution: ~226 µs
  • Post-processing: Python (histogram analysis, false-trigger filtering)

What I Did

  • Designed and implemented real-time C++ firmware on the Bela platform to synchronously sample accelerometer and audio signals
  • Developed tap-detection logic with false-trigger filtering to isolate genuine touch events from noise
  • Designed and 3D-printed a custom sensor mount in FreeCAD to ensure repeatable, consistent tap events across test runs
  • Post-processed raw timestamp data in Python, generating latency histograms and statistical summaries across multiple devices
